# Add Graphite stack-review runbook to workflow pack

## Scope

Add a new rule module `rules/graphite-stack-review.md` (\~60–80 lines) capturing the runbook for handling reviewer comments on a Graphite-stacked PR series — surfaced during [ABA-299](https://linear.app/ababushkin/issue/ABA-299/resume-a-halted-issue-on-re-run-reuse-the-preserved-worktree)'s 4-PR stacked delivery, which had no written home anywhere in the pack.

Style matches `rules/linear-workflow.md`: second person, numbered procedure sections, backtick-wrapped commands inline.

Content sections, in order:

1. **Fetch the reviewer's comments.** `gh api repos/{owner}/{repo}/pulls/{N}/comments` for each PR in the stack — quote the exact command form because it's the load-bearing primitive.
2. **Triage by layer.** Identify which PR/layer owns each comment (a comment on PR 3 about a line introduced in PR 1 still gets fixed in PR 1).
3. **Amend bottom-up.** `gt checkout <bottom-branch>`, make the fix, `git commit --amend --no-edit`. Stay on that layer until committed.
4. **Propagate with restack.** `gt restack` to walk parent SHAs upstream so every higher layer points at the new bottom-layer commit.
5. **Force-push the stack.** `gt submit --stack` pushes all amended layers in one shot. Verify with `gh pr view {N} --json headRefOid` that the GitHub head SHA matches local.

Also: add a one-line pointer to the new module from the modules-index area of `rules/GOVERNANCE.md`, in the same style as the existing `Read rules/linear-workflow.md for …` pointer. Bump `version` in `.claude-plugin/plugin.json`.

## Acceptance criteria

* `rules/graphite-stack-review.md` exists with the five numbered sections above.
* `grep -n "graphite-stack-review" rules/GOVERNANCE.md` returns exactly 1 line.
* `jq -r .version .claude-plugin/plugin.json` is higher than current main.
